# Vision-тест для "Читаем вместе"

## Что это

Интерактивный тест, где **Playwright** делает скриншоты, а **LLM (Qwen Vision)** анализирует экран и решает куда кликать.

## Запуск

```bash
node vision-test/interactive.js
```

## Команды во время теста

| Команда | Описание |
|---------|----------|
| `click 0` | Кликнуть на карточку 0 |
| `click 1` | Кликнуть на карточку 1 |
| `click 2` | Кликнуть на карточку 2 |
| `click 3` | Кликнуть на карточку 3 |
| `next` | Кликнуть "Дальше" |
| `anomaly: текст` | Записать аномалию |
| `screenshot` | Сделать скриншот |
| `quit` | Завершить тест |

## Как работает цикл LLM-VISION

1. Playwright открывает игру, делает скриншот
2. Скриншот отправляется Qwen Vision
3. Qwen анализирует: какое слово, какие картинки, где правильная
4. Qwen даёт команду: `click 2` (правильная) или `click 0` (ловушка)
5. Playwright кликает → новый скриншот
6. Qwen проверяет обратную связь (✅/❌, подсветка)
7. Записывает аномалии если нашёл баг

## Что ищем

- ❌ Неправильные картинки (слово "кот", картинка "кит" как правильная)
- ❌ Ловушки не работают (все 4 картинки одинаковые)
- ❌ UI баги (кнопка обрезана, текст не читается)
- ❌ Логика сломана (правильный ответ не подсвечивается зелёным)

## Артефакты

- `screenshots/` — скриншоты каждого шага
- `anomalies.json` — найденные баги с путями к скриншотам
